# Study smoke-test artifacts

Visual + machine-readable evidence for the smoke-test pass over the 12 user-
study images. Generated by [`backend/scripts/render_study_artifacts.py`](../../backend/scripts/render_study_artifacts.py)
and [`backend/scripts/validate_study_analyses.py`](../../backend/scripts/validate_study_analyses.py).

## What's here

```
docs/study_smoke_test/
├── README.md                  ← you are here
├── spotcheck.md               ← hand-spot-check log, fill in per provider
└── <id>_<filename>/           ← one folder per image (12 total after a full run)
    ├── original.jpg
    ├── ela_overlay.png        ← same overlay sent to the VLM
    ├── forensic_strip.png     ← per-region z-score bar chart
    └── metrics.json           ← raw + z-scored numbers (for the spotcheck)
```

## How to regenerate

From the repo root with the backend venv active and the `.env` populated:

```bash
# 1. Bring up the FastAPI backend so models load from HF Hub on first call.
cd backend
uvicorn app.main:app --reload --port 8000

# In a second terminal:
# 2. Run precompute over the 12 study images for all four providers.
#    (Hits the live OpenAI / Google / Anthropic APIs — costs ~$0.50–$2 total.)
curl -X POST http://localhost:8000/api/v1/study/precompute

# 3. Validate the resulting JSON.
python backend/scripts/validate_study_analyses.py

# 4. Render the visual smoke-test artifacts (no VLM calls — pure forensics).
python backend/scripts/render_study_artifacts.py

# 5. Open spotcheck.md and fill in the per-provider notes by reading the
#    rendered ELA overlays + forensic strips alongside the explanations in
#    desktop/public/study-analyses.json.
```

## Acceptance gate

The smoke test passes when:

1. `validate_study_analyses.py` exits 0 — every cell of the 12 × 4 matrix
   has `structured_regions` populated with a non-empty `evidence_ref` per
   region (or the provider returned a clean error and is in `--allow-error`).
2. Every image has its `<id>_<filename>/` folder rendered with the four
   files listed above.
3. The spot-check log in `spotcheck.md` records at least 4 hand-reviewed
   explanations (one per provider) and flags any obvious confabulation.

After all three pass, the precomputed assets in `desktop/public/` are safe
to bundle into the user-study build (Issue 4).
